# AI 주식 추천 시스템 - MVP 아키텍처 가이드라인

## 1. 프로젝트 개요
이 프로젝트는 사용자가 입력한 주요 주식 종목에 대해 Supabase DB에 적재된 데이터와 Anthropic Claude API를 연동하여 최적의 매수/매도 타이밍 리포트를 실시간으로 제공하는 AI 기반 금융 MVP 시스템입니다. Vercel 인프라 환경(Hobby 플랜 및 서버리스 런타임)에 최적화하여 배포됩니다.

## 2. 기술 스택 및 인프라 제약 조건
- **프론트엔드 프레임워크:** React (Vite 기반 SPA 아키텍처)
  - 초기 HTML은 `<div id="root"></div>` 구조로 렌더링되며, 이후 브라우저 실행 단계(클라이언트 사이드)에서 자바스크립트가 UI 컴포넌트와 주식 대시보드를 동적으로 그려내도록 설계할 것.
- **데이터베이스 및 백엔드:** Supabase (PostgreSQL) 및 Vercel 서버리스 함수 (Serverless Functions)
  - **서버리스 타임아웃 방어 규칙:** Vercel 무료 티어의 실행 시간 제한(Execution Timeout)을 고려하여, 무거운 주가 데이터 스크래핑이나 대량의 LLM 연산은 실시간 요청 파이프라인에서 분리하고 Supabase의 캐시 데이터 구조를 적극 활용할 것.
- **AI 엔진:** Anthropic Claude API (콘솔 종량제 계정 아키텍처 연동)

## 3. 보안 거버넌스 및 컴플라이언스 지침 (필수 준수)
- **API 키 및 인증 정보 보호 (통제항목 2.10.2 암호 키 관리):**
  - `ANTHROPIC_API_KEY`, Supabase 접속 비밀번호 등 주요 인증 정보 및 시스템 자산 키는 소스코드 내부(클라이언트 스크립트 포함)에 절대 직접 하드코딩(Hard-coding)하지 말 것.
  - 반드시 Vercel 환경 설정(Dashboard > Settings > Environment Variables) 및 로컬 환경의 `.env` 파일을 통해서만 동적으로 호출되도록 통제할 것.
- **개발·시험·운영 환경 격리 (통제항목 2.3.2 분리):**
  - 모든 코드 수정과 기능 실험은 로컬 개발 환경(`localhost:5173`) 내에서 검증을 선행하며, 완전 무결성이 증명된 결과물만 Vercel 프로덕션 파이프라인으로 이관 배포할 것.

## 4. 주요 개발 명령어
- **로컬 개발 서버 실행:** `npm run dev` (localhost:5173으로 구동)
- **프로젝트 빌드:** `npm run build`
- **Vercel 프로덕션 배포:** `vercel --prod`
- **의존성 패키지 통제:** 일회성 패키지 실행 및 오픈소스 공급망 취약점 사전 방지를 위해, 패키지의 로컬 영구 설치를 최소화하고 `npx` 임시 실행 특공대를 적극 활용할 것.
